    <article class="markdown-body entry-content" itemprop="text"><h1><a href="#coding-challenge" aria-hidden="true" class="anchor" id="user-content-coding-challenge"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Coding Challenge</h1>
<h2><a href="#problem-description" aria-hidden="true" class="anchor" id="user-content-problem-description"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Problem Description</h2>
<p>Blockchain is the new technology of the art. With a view of powering the well known Bitcoin cryptocurrency, new blocks have to be mined into the blockchain. In this challenge, the mining task should be tackled in a distributed manner on smartphones.</p>
<p>Bitcoin mining works in the way that strong mathematical puzzles have to be solved for a proof of work, more specifically calculating a hash of the block's header that is to mine with a certain number of leading zeros (difficulty). If the hash has enough leading zeros, the challenge is solved. Otherwise, the block headerâs nuance is altered and a new hash is calculated.</p>
<h2><a href="#challenge-overview" aria-hidden="true" class="anchor" id="user-content-challenge-overview"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Challenge Overview</h2>
<p>Implement a distributed Bitcoin mining application with following criteria:</p>
<ul>
<li>Create a central web service that delegates and aggregates the work to/from the clients. Use NodeJS to implement this logic and deploy it on a server (e.g., Amazon AWS).</li>
<li>Implement the client hashing logic as an Android app that is responsible for doing the hard calculations in a distributed manner. The client app gets the puzzles to solve from the server and publishes its results to it.</li>
<li>Follow the REST principles for the communication between client and server. The endpoints could look like this, for instance:
<ul>
<li><code>/work</code>   - returns a JSON-encoded job that the client shall use to begin mining, e.g.:
<pre><code>{
	jobId: 2
	clientId: 5,
	blockHeader: {
		version: 2,
		prevBlockhash: 00000000000008a3a41b85b8b29ad444def299fee21793cd8b9e567eab02cd81,
		merkleRoot: 2b12fcf1b09288fcaff797d71e950e71ae42b91e8bdb2304758dfcffc2b620e3,
		timestamp: 1305998791,
		difficultyTarget: 17,
		Nonce: 2504433986
	}
}
</code></pre>
</li>
<li><code>/submit</code> - provides the client with the possibility to submit a successful nonce to the server or when he finished his assigned task.</li>
</ul>
</li>
<li>Use an appropriate authentication mechanism.</li>
<li>Realize the client computations in the most performant way you know.</li>
<li>You can create pseudo-hashing-puzzles and do not have to connect to the real bitcoin blockchain.</li>
</ul>
<h2><a href="#extensions" aria-hidden="true" class="anchor" id="user-content-extensions"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Extensions</h2>
<p>Feel free to pick from the following extensions to extend your solution, but they are not required:</p>
<ul>
<li>Use Kotlin instead of Java.</li>
<li>Use TypeScript instead of pure JavaScript.</li>
<li>Connect your server to the bitcoin blockchain to retrieve ârealâ block data.</li>
</ul>
